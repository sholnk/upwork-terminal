## 4. Profile Structure

### Basic Information
| Field | Max Length | Description |
|-------|-----------|-------------|
| Profile Photo | - | 推奨サイズ 450x450px |
| Name | - | 本名 |
| Title | 70文字 | 職種タイトル |
| Overview | 5000文字 | 自己紹介・経歴 |
| Location | - | 国・都市 |
| Timezone | - | タイムゾーン |

### Skills
* 最大15スキル
* UpWork標準リストから選択
* 並び順が検索に影響

### Rate & Availability
| Field | Options |
|-------|---------|
| Hourly Rate | USD金額 |
| Weekly Hours | Less than 30 / More than 30 / Both |
| Availability Badge | ON/OFF (Connects消費) |

### Portfolio
* 作品タイトル
* 説明文
* 画像/動画
* 外部リンク
* 関連スキル

### Work History
* 会社名
* 役職
* 期間（開始〜終了）
* 説明

### Education
* 学校名
* 学位/専攻
* 期間

### Certifications
* 資格名
* 発行機関
* 取得日

## 5. GraphQL API Structure

### Main Queries
```graphql
# プロフィール取得
query {
  talentProfileByProfileKey(profileKey: "xxx") {
    identity { ... }
    personalData { ... }
    preferences { ... }
  }
}

# ユーザー情報取得
query {
  user {
    id
    name
    email
    profiles { ... }
  }
}
```

### Profile Fragments
* `identity` - ID情報
* `personalData` - 個人情報
* `preferences` - 設定情報
* `skills` - スキル一覧
* `portfolio` - ポートフォリオ
* `employmentHistory` - 職歴
* `educationHistory` - 学歴

## 6. OAuth 2.0 Flow

```
1. Authorization Request
   GET https://www.upwork.com/ab/account-security/oauth2/authorize
   ?client_id={CLIENT_ID}
   &redirect_uri={REDIRECT_URI}
   &response_type=code
   &scope=read%20write

2. User Authorization (ブラウザ)

3. Authorization Code Callback
   GET {REDIRECT_URI}?code={AUTH_CODE}

4. Token Exchange
   POST https://www.upwork.com/api/v3/oauth2/token
   {
     "grant_type": "authorization_code",
     "code": "{AUTH_CODE}",
     "client_id": "{CLIENT_ID}",
     "client_secret": "{CLIENT_SECRET}",
     "redirect_uri": "{REDIRECT_URI}"
   }

5. API Request
   GET https://api.upwork.com/graphql
   Authorization: Bearer {ACCESS_TOKEN}
```

### Scopes
* `read` - 読み取り専用
* `write` - 読み書き

## 8. Best Practices

### Profile Optimization
* タイトルは具体的かつニッチに
* Overviewはクライアント視点で記述
* 上位5スキルを慎重に選択
* ポートフォリオは定期更新

### API Usage
* レート制限を考慮したバッチ処理
* トークンの自動リフレッシュ
* エラーハンドリングとリトライ
* ローカルキャッシュ活用
